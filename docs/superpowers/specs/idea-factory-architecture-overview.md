## 层间交互原则

1. **控制面 ↔ 执行协议层**
   - 控制面通过结构化控制动作影响执行
   - 执行协议层通过事件流反馈控制面
   - 所有用户干预必须转换为可追溯的控制动作

2. **执行协议层 ↔ 长期能力层**
   - 执行协议层提供运行上下文给长期能力层
   - 长期能力层提供记忆、技能和偏好给执行协议层
   - 长期能力持久化影响未来运行行为

3. **所有层 ↔ 核心引擎**
   - 控制面提供治理约束
   - 执行协议层提供安全边界和执行协议
   - 长期能力层提供记忆和技能资源
   - 核心引擎在三层约束下进行图谱生长决策

## 关键接口

- **控制动作接口**：结构化的治理命令（JSON schema）
- **执行协议接口**：run/turn/checkpoint生命周期管理
- **记忆接口**：持久化和检索的工作记忆和用户偏好
- **技能接口**：发现、绑定和执行的探索技能
- **工具接口**：受控的外部能力访问（研究、文件操作等）

## 演进路径

### 第一阶段：基础三层建立
- 建立基本的控制面（探索控制台）
- 实现执行协议层（run-turn-checkpoint协议）
- 建立长期能力层基础（简单记忆存储）

### 第二阶段：能力增强
- 增强控制面（审查、对比、产物生成）
- 完善执行协议层（工具风险策略、审批流程）
- 开发长期能力层（技能系统、用户偏好学习）

### 第三阶段：平台开放
- 开放API/SDK供扩展
- 支持专家系统和多入口协作
- 完善插件和适配器机制

## 与参考系统的对应关系

| Idea Factory 层 | 对应参考系统 | 核心继承概念 |
|----------------|-------------|--------------|
| 控制面 (L1)    | Claude Code | 显式控制表面、命令系统、模式切换、治理入口 |
| 执行协议层 (L2)| Codex       | 任务-回合协议、工具风险策略、检查点机制、不可变追踪 |
| 长期能力层 (L3)| Hermes Agent| 持久会话记忆、技能系统、用户偏好模型、跨运行上下文 |

这种架构确保了：
- 用户能够清晰地理解和控制系统行为（控制面）
- 系统执行安全、可追溯且可恢复（执行协议层）
- 系统随时间学习和改进而不丢失进度（长期能力层）
